# Product 文档入口

本目录放产品侧的长期参考材料。这里的文档回答“为什么做、为谁做、长期想做到什么”，不直接作为当前实现状态的最终判断。

日常开发时，功能是否已经落地以 `../05_Current_Feature_Status_Matrix.md` 为准；技术实现规则以 `../02_Developer_SOP_and_Guide.md` 和 `../09_Architecture_Overview.md` 为准。

## 当前产品基线

- 产品定位：AI 投资分析与组合辅助系统。
- 核心问题：把分散的持仓、行情、宏观、新闻和 AI 分析整理成可行动、可追踪、可复盘的判断辅助。
- 当前阶段：优先支持 Web、FastAPI 后端、多源行情、AI Provider/BYOK、通知和后台调度。
- 明确不做：自动实盘交易系统；AI 输出不作为投资承诺。

## 文档

| 文档 | 作用 | 使用方式 |
|------|------|----------|
| `01_Product_Requirements_Document.md` | 完整 PRD，包含产品愿景、用户研究、功能规划、技术设想、商业化和风险 | 产品范围、路线图、商业化和长期规划讨论时参考 |

## 阅读建议

1. 如果你只是要改代码，先读 `../README.md`、`../05_Current_Feature_Status_Matrix.md`、`../02_Developer_SOP_and_Guide.md`。
2. 如果你要判断“这个功能该不该做、优先级如何”，再读完整 PRD。
3. 如果完整 PRD 和当前代码或状态矩阵冲突，以代码实际行为和状态矩阵为准，并在同一轮变更中修正文档。
