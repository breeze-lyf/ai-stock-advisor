---

## 1. 使用说明

本清单用于回答三个问题：

1. 当前这个功能是否已经落地。
2. 当前是“已完成”还是“部分完成”。
3. 研发提交后，测试最少要验证哪些路径。

状态定义：

- 已完成：主流程可稳定使用，具备基础异常处理。
- 部分完成：核心链路可用，但生命周期或边界能力未齐。
- 未开始：仓库中暂无可运行实现。

| 检查项 | 要求 | 是否阻塞提测 |

## 5. 运行与环境核查清单

| 项目 | 核查内容 | 备注 |
|------|----------|------|
| 后端环境变量 | `DATABASE_URL`、`SECRET_KEY`、AI Provider Key | 缺失会阻塞主链路 |
| 数据库 | Neon/PostgreSQL 可连接，迁移已到 `head` | 禁止手改线上表结构 |
| Redis | 可连通，用于通知去重与缓存 | 非主存储但影响体验 |
| 调度器 | 启动后是否正常拉起并写日志 | 影响宏观与摘要类功能 |
| 前端环境变量 | `NEXT_PUBLIC_API_URL` 正确 | 影响所有 API 调用 |

## 6. 版本维护规则

1. 每次功能状态发生变化（部分完成 -> 已完成）时，必须同步更新本矩阵。
2. 每次提测如涉及跨模块改动，测试负责人应在本矩阵追加本轮重点验证项。
3. 本文档与 PRD、README、SOP 冲突时，以代码实际行为为准，并在同一迭代内统一修正。
